# Android 音视频处理应用任务提示

## 目标
开发一个 Android 应用程序，使用手机摄像头和麦克风获取音视频流，使用 `GLSurfaceView` 将视频画面渲染到整个屏幕上，并在底部提供一个包含三个功能按钮的菜单栏：“直方图”、“波形图”和“单色”。

## 需求

### 1. 音视频采集
- 使用 Android 的 `Camera2` API（如果为了简化也可以使用 `CameraX`，但在 OpenGL 中处理原始数据时 `Camera2` 通常更好）来捕获视频帧。
- 使用 `AudioRecord` 或 `MediaRecorder`（如果将来需要处理原始数据，`AudioRecord` 更好）来捕获音频。*注意：提示主要强调视频处理功能，但也要求了音频捕获。*
- 确保正确处理摄像头和麦克风的权限（运行时权限）。

### 2. 渲染
- 使用 `GLSurfaceView` 将摄像头预览渲染到整个屏幕。
- 实现自定义的 `GLSurfaceView.Renderer`。
- 使用 OES 纹理 (`GLES11Ext.GL_TEXTURE_EXTERNAL_OES`) 高效处理摄像头预览帧。

### 3. UI/UX
- 视频预览应充满整个屏幕。
- 底部菜单栏应覆盖在视频底部的上方。
- 菜单应包含三个按钮：
    1.  **Histogram** (直方图)
    2.  **Waveform** (波形图)
    3.  **Monochrome** (单色)

### 4. 功能
- **Monochrome (单色滤镜)**：
    - 点击时，对视频流应用基于着色器（Shader）的滤镜。
    - 效果应类似于摄影中的“灰度”或特定的单色色调。
    - 这应在片元着色器（Fragment Shader）中实现。
- **Histogram & Waveform (直方图与波形图)**：
    - 这些按钮应切换*当前屏幕亮度信息*的可视化显示。
    - **Histogram (直方图)**：显示亮度直方图叠加层。这需要计算帧的亮度分布。
    - **Waveform (波形图)**：显示亮度波形图叠加层（通常 x 轴是图像 x 位置，y 轴是亮度强度）。
    - 这些可视化效果最好使用 OpenGL 渲染（例如，在视频上方绘制线条/点）或在 GLSurfaceView 之上使用自定义 View。鉴于“渲染到 GLSurfaceView”的要求，在 GL 中实现更高效且符合视频应用的标准。

### 5. 代码质量与架构
- **语言**：Kotlin（现代 Android 开发首选）或 Java。*我将使用 Kotlin（实际代码使用了Java）。*
- **架构**：MVVM 或清晰的关注点分离。
- **风格**：严格遵守 Google 的 Android 编码规范。
- **可扩展性**：滤镜和分析管道的设计应易于添加新滤镜或分析工具。
- **健壮性**：正确处理生命周期事件（onPause, onResume）以防止崩溃或电池耗尽。

## 开发步骤
1.  **项目设置**：在 `AndroidManifest.xml` 中配置权限和 OpenGL 版本。
2.  **着色器实现**：编写顶点和片元着色器，用于：
    - 基本 OES 纹理渲染。
    - 单色/灰度滤镜。
    - （可选）用于直方图/波形图计算的计算着色器或高效逻辑。
3.  **OpenGL 渲染器**：实现 `GLSurfaceView.Renderer` 以管理纹理、着色器和绘制。
4.  **摄像头设置**：实现一个辅助类来管理 `Camera2` 会话并将其连接到 OpenGL 纹理。
5.  **UI 实现**：创建包含 `GLSurfaceView` 和底部按钮栏的布局。
6.  **逻辑集成**：将按钮连接到渲染器以切换着色器或启用叠加层。
